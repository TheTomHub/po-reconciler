Installation (2 minutes)

Chandlr is an Excel add-in. You install it once, then it appears in your ribbon.

Option A: Excel Desktop (Windows/Mac)

1
Open Excel and go to Insert → Office Add-ins

In the ribbon at the top, click the Insert tab, then click Office Add-ins (or "Get Add-ins").

2
Click "Upload My Add-in"

In the dialog, click Upload My Add-in in the top-right corner. If prompted, choose Browse.

3
Select the manifest file

Download manifest.xml and select it. Click Upload.

4
Find "Chandlr" in the ribbon

A new Chandlr tab appears in the ribbon. Click Open Chandlr to launch the task pane.

One-time setup Once installed, Chandlr stays in your ribbon. You don't need to upload the file again.

Option B: Excel for the Web

1
Open a workbook in Excel Online

Go to office.com/launch/excel and open any workbook.

2
Insert → Office Add-ins → Upload My Add-in

Same steps as desktop. Upload the manifest.xml file.

The Workflow

Chandlr takes you through a linear pipeline. Each step feeds into the next:

Upload PO → Extract & Validate → Load ERP → Reconcile → Actions → ERP Staging

The task pane has three tabs: Reconcile (steps 1–4), Actions (steps 5–7), and Tools (price intelligence). With the Copilot agent, steps 2–3 can be automated via SharePoint — see Using the Copilot Agent.

Step 1: Upload a PO

Click Choose File and select the customer PO. Supported formats: .csv, .xlsx, .xls, .pdf

Step 6: Take Action

Switch to the Actions tab. Depending on the results, you'll see these buttons:

Chandlr — Actions Tab
Draft Exception Email Generate Credit Note Generate Re-Invoice Generate ERP Staging

Draft Exception Email

Creates an email draft summarizing the top pricing exceptions. You can copy the text or open it directly in your email client.

Generate Credit Note

Creates a CreditNote sheet that credits ALL PO lines at their original PO prices (as negative amounts). This is company policy: credit everything, then re-invoice at correct prices.

Generate Re-Invoice

Creates a ReInvoice sheet with corrected ERP prices for all lines. Rows where the price changed are highlighted yellow.

Credit Note + Re-Invoice buttons only appear when there are exceptions. If all prices match, you don't need them.

Step 7: Generate ERP Staging

Click Generate ERP Staging to create a clean sheet ready for ERP entry. This is the final output — the data you'll enter into SAP, Oracle, D365, or whatever system you use.

Using the Copilot Agent

If your organisation has Microsoft 365 Copilot, Chandlr works as a Copilot agent — you can do everything by typing natural language instead of clicking buttons.

Requirements Microsoft 365 Copilot license (or pay-as-you-go billing enabled). The agent uses SharePoint to find price lists automatically.

SharePoint integration

Chandlr can access price lists and reference data stored in your team's SharePoint folders. This means:

  • No manual copy-paste — the agent finds the latest price list automatically
  • Always up-to-date — if your team updates the price list daily, the agent uses today's version
  • Works with existing workflows — no need to change where you store files
Tip: be specific about which price list If your team has multiple price lists (e.g. by supplier or category), tell the agent: "Use the Tesco price list from the Pricing folder" to help it find the right one.

Price Intelligence (the AI moat)

Every reconciliation builds a price history. The Copilot agent analyses this history to surface insights you'd otherwise miss:

  • Risk assessment — after reconciliation, the agent automatically assesses whether a PO should be accepted, reviewed, or escalated. It factors in exception rate, exposure, and historical patterns.
  • SKU lookup — ask about any product and get its full price timeline, trend direction (up/down/stable), volatility, and risk flags.
  • Anomaly detection — prices that fall outside their historical range are flagged automatically. No manual analysis needed.
  • Watch list — SKUs with recurring issues (frequent exceptions, large changes, high volatility) are tracked and highlighted.
  • Conversational analysis — unlike a static dashboard, you can ask follow-up questions: "Why is SKU 1046 on the watch list?" or "Which supplier has the most anomalies?"
The more you reconcile, the smarter it gets. Each reconciliation adds data points. After 3–5 runs, you'll start seeing meaningful trends and the risk assessments become more accurate.

Tips & Troubleshooting

Column detection isn't working

If Chandlr can't auto-detect your columns, it shows dropdowns. Make sure your header row uses recognizable names like "SKU", "Price", "Qty", "Description", etc. Rename ambiguous headers if needed.

ERP data won't load

Make sure you've selected the data range in Excel before clicking "Load Selection". The selection must include the header row.

SKUs don't match between PO and ERP

Chandlr handles common mismatches automatically:

  • Prefix matching: PO SKU "1046" will match ERP SKU "1046V001"
  • Case-insensitive: "abc123" matches "ABC123"
  • Whitespace: Leading/trailing spaces are trimmed

If a SKU still shows "Not in ERP", it likely needs to be set up in your ERP system.

The task pane is empty or shows an error

Try: close the task pane (X), then reopen it from the Chandlr ribbon tab. If that doesn't work, remove and re-upload the add-in.

Changing the tolerance

Adjust the tolerance value (in the Settings section) before running reconciliation. For example, set it to 0.05 if you're OK with up to 5p difference.

Changing the currency

Use the currency dropdown (GBP/USD/EUR) in Settings. This affects formatting only — it doesn't convert values.
