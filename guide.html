Three ways to use Chandlr

Same pipeline, any surface Extract → Validate → Reconcile → Resolve → Stage → Predict. Whether you use the web app, Teams, or Excel, every reconciliation feeds the shared price intelligence layer.

Teams Agent workflow

If your organisation has Microsoft 365 Copilot, you can process POs entirely inside a Teams chat. No Excel, no file browsing — just attach and prompt.

Requirements Microsoft 365 Copilot licence (or pay-as-you-go Copilot billing). The Chandlr app must be installed by your Teams admin.
Attach PO → Prompt Chandlr → SharePoint fetch → Reconcile → Outputs in chat

Step by step

1
Open Chandlr in Teams

Find Chandlr in the sidebar under "Apps". If it's not there, search for it in the Teams app store or ask your admin to install it.

2
Attach the PO and say "Process this PO end to end"

Click the paperclip icon in the chat input, attach the PO file (PDF, CSV, or Excel), and send the prompt. Chandlr accepts any format — it handles company headers, merged cells, and non-standard layouts automatically.

3
Chandlr extracts the PO and searches SharePoint

It reads the customer name from the PO, searches your SharePoint and OneDrive for the latest matching price list, and confirms the file with you before proceeding.

4
Review the reconciliation results and risk assessment

Chandlr returns a summary: total lines, matches, exceptions, and total exposure. It then automatically runs a risk assessment and recommends ACCEPT, REVIEW, or ESCALATE.

5
Say "Generate outputs" if exceptions exist

Chandlr generates a credit note and re-invoice for the exception lines only.

6
Say "Email and staging" to finalise

Chandlr drafts the exception email and generates an ERP-ready staging sheet. A summary of all outputs is presented at the end.

Excel Add-in: Installation

Chandlr is a free Excel add-in. You install it once, then it appears in your ribbon.

Option A: Excel Desktop (Windows/Mac)

1
Open Excel and go to Insert → Office Add-ins

In the ribbon at the top, click Insert, then Office Add-ins (or "Get Add-ins").

2
Click "Upload My Add-in"

In the dialog, click Upload My Add-in (top-right). If prompted, choose Browse.

3
Select the manifest file

Download manifest.xml and select it. Click Upload.

4
Find "Chandlr" in the ribbon

A new Chandlr tab appears. Click Open Chandlr to launch the task pane.

One-time setup Once installed, Chandlr stays in your ribbon. You don't need to upload the file again.

Option B: Excel for the Web

1
Open a workbook in Excel Online

Go to office.com/launch/excel and open any workbook.

2
Insert → Office Add-ins → Upload My Add-in

Same steps as desktop. Upload manifest.xml.

Excel Add-in: The Workflow

Chandlr takes you through a linear pipeline. Each step feeds into the next:

Upload PO → Extract & Validate → Load ERP → Reconcile → Actions → ERP Staging

The task pane has four tabs: Reconcile, Actions, Tools, and Prompts. The Prompts tab shows suggested Copilot prompts — click any to copy it, then paste into the Copilot panel. With the Copilot agent active in Excel, the entire pipeline runs from a single prompt and the price list is fetched from SharePoint automatically (skip Step 3). See Using Copilot in Excel.

Click Choose File and select the customer PO. Supported formats: .csv, .xlsx, .xls, .pdf

  2. Normalize SKUs, prices, quantities, and descriptions
  3. Write a clean "PO <ref>" sheet in your workbook
  4. Run 9 validation checks on the data

Step 3: Load ERP Data

Using the Copilot agent in Excel? Skip this step. After extraction, the agent automatically searches SharePoint for the latest price list matching the account name and writes it to an ERPPrices sheet. Reconciliation then runs against that sheet — no manual selection needed.

If you are using the task pane (button mode), load your ERP price list manually:

A
Open or paste your ERP data into a sheet

This should have at least a SKU column and a Price column. Paste it into a new sheet in the same workbook, or open it alongside.

B
Select the entire ERP data range

Click the top-left cell and drag to select all rows and columns, including the header row.

C
Click "Load Selection" in Chandlr

Back in the task pane, click Load Selection under step 2. You'll see the range address and detected column names.

Important: select the header row too Your selection must include the header row. Chandlr uses the headers to auto-detect which column is which.

SharePoint integration

Chandlr has access to your team's SharePoint and OneDrive. After extracting a PO it automatically:

  1. Reads the customer name from the PO
  2. Searches SharePoint for the latest price list file matching that account
  3. Asks you to confirm the file before writing it
  4. Writes the price data to an ERPPrices sheet and runs reconciliation against it
Tip: be specific if needed The agent searches by account name automatically. If you have multiple price lists (e.g. by brand), say "Use the Hydroflask price list" to narrow it down.

Tips & Troubleshooting

Column detection isn't working

If Chandlr can't auto-detect your columns, it shows dropdowns. You can also open the Chandlr task pane and use the manual column selector in Step 1. You do not need to reformat or restructure your spreadsheet.

ERP data won't load

If using the Copilot agent: ERP data is loaded automatically from SharePoint. If the agent can't find a matching price list, it will tell you and ask you to select the range manually.

If using the task pane: make sure you've selected the data range in Excel (including the header row) before clicking "Load Selection".

SKUs don't match between PO and ERP

Chandlr handles common mismatches automatically:

  • Prefix matching: PO SKU "1046" matches ERP SKU "1046V001"
  • Case-insensitive: "abc123" matches "ABC123"
  • Whitespace: Leading/trailing spaces are trimmed

If a SKU still shows "Not in ERP", it likely needs to be set up in your ERP system.

The task pane is empty or shows an error

Try: close the task pane (X) and reopen from the Chandlr ribbon tab. If still blank, hard-refresh (Cmd+Shift+R on Mac / Ctrl+Shift+R on Windows). If that doesn't help, remove and re-add the add-in via Insert → Add-ins.

Changing the tolerance

Open the Settings panel (gear icon in the task pane header) and adjust the tolerance value before running reconciliation. For example, 0.05 allows up to 5p difference.

Changing the currency

Use the currency dropdown (GBP/USD/EUR) in the Settings panel. This affects formatting only — it doesn't convert values.

The price intelligence is empty

The intelligence layer needs at least one completed reconciliation to have data. After 3–5 runs you'll start seeing meaningful trends and the ACCEPT/REVIEW/ESCALATE recommendations become more accurate.
